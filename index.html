Dashboard de Consommation Énergétique

WinForms • C# • Windows • Transition Énergétique

🚀 Projet Démarrage 21 Janvier 2026

📋 Contexte & Commanditaire

🌍 Contexte du Projet

L'entreprise fictive vise la transition énergétique et la réduction de l'empreinte carbone en centralisant les données énergétiques dispersées dans différents systèmes.

🎯 Mission Principale

Concevoir une application Windows en C# avec Windows Forms pour centraliser et visualiser les données énergétiques en temps réel avec une interface ergonomique.

✅ Objectifs Clés

Offrir un outil simple, performant et adapté à Windows sans technologies externes, capable de détecter les anomalies de consommation automatiquement.

Enjeux & Motivations

Réduire l'empreinte carbone de 25 kWh/an, améliorer la visibilité énergétique et permettre une prise de décision rapide face aux anomalies détectées.

Détails des Objectifs

🔄 Centralisation des Données

Centraliser les données provenant de capteurs physiques ou sources simulées pour une gestion efficace et une visibilité complète.

📈 Affichage Graphique Clair

Les données énergétiques sont présentées sous forme de graphiques et indicateurs lisibles via Windows Forms.

🔍 Analyse des Consommations

Détection des pics de consommation grâce à des seuils prédéfinis pour une meilleure analyse et intervention rapide.

🖥️ Interface Ergonomique

Interface intuitive et compatible avec Visual Studio pour une utilisation simple et pédagogique sous Windows.

⚙️ Fonctionnalités Clés (WinForms / 100% C#)

📡 Collecte de Données Locale

Les données sont collectées via une API ASP.NET Core en C# hébergée localement avec mises à jour régulières programmées.

🧮 Agrégation et Calcul

Le traitement comprend des agrégations simples par minute, heure et jour, ainsi que le calcul du kWh total consommé.

⚡ Détection d'Anomalies

Détection d'anomalies utilisant des seuils fixes pour identifier rapidement les écarts anormaux dans les données collectées.

🎨 Interface Graphique Simple

L'interface Windows Forms intègre un graphique, un tableau des dernières valeurs et des indicateurs visuels pour les alertes.

📂 Projets Visual Studio

La solution comprend deux projets clés : WinForms pour l'interface utilisateur et ASP.NET Core pour l'API de données.

🎯 Composants WinForms

Le projet WinForms gère les formulaires, services métier et modèles de données pour l'interface utilisateur cohérente.

Architecture de l'API ASP.NET Core

Le projet API contient les contrôleurs, modèles et services pour la gestion et collecte des données énergétiques.

Base de Données SQLite Locale

SQLite est utilisé pour un stockage léger et local, géré via Entity Framework Core pour la simplicité et maintenabilité.

👥 Personas Utilisateurs

👨‍💼 Persona 1 - Marc, Technicien d'Exploitation (45 ans)

Contexte : Technicien polyvalent, gère plusieurs bâtiments. Besoin d'un outil rapide pour repérer des pics et agir.

Objectifs : Visualiser les valeurs instantanées, être alerté en cas de dépassement, historiser un minimum.

Points de douleur : Interfaces confuses, lenteurs, données incohérentes.

Scénario d'usage : Marc ouvre l'app le matin, contrôle la conso de la nuit, détecte un pic anormal et planifie une intervention.

👩‍💼 Persona 2 - Élodie, Responsable Énergie / RSE (38 ans)

Contexte : Responsable de la performance énergétique, prépare des bilans mensuels.

Objectifs : Suivre les tendances mensuelles, vérifier la trajectoire de réduction, comparer des périodes.

Points de douleur : Données dispersées, exports multiples, manque de lisibilité synthétique.

Scénario d'usage : Élodie consulte le tableau de bord mensuel, identifie une dérive en octobre, et demande une analyse des causes.
